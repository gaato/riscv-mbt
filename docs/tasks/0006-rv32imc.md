# Task 0006: Implement RV32IMC

## Background

`RV32IMC` is the “core completion” milestone where compressed instructions make fetch and decode materially more complex.

## Work

- Keep the new length-aware fetch path as the boundary for mixed-width instruction streams
- Use 16-bit instruction length detection to branch between compressed and 32-bit fetch/decode paths
- Expand compressed instructions into the existing internal `Instruction` forms instead of adding a separate compressed execute pipeline
- Extend the current compressed subset beyond the initial arithmetic/jump/load-store/SP forms to broader `RV32IMC` coverage, including the reusable integer ALU/immediate forms
- Add mixed 16/32-bit program coverage
- Keep official `rv32uc` execution survey-only until the remaining reserved/hint/unimplemented compressed forms are closed
- Use the survey result to decide when official compressed coverage is broad enough to promote beyond survey-only use

## Acceptance Criteria

- `pc` updates remain correct across mixed instruction widths
- Compressed and non-compressed instruction streams can coexist in integration tests
- Built upstream `*-p-*` survey results show `rv32ui`, `rv32um`, and the current `rv32uc` coverage passing on the emulator
